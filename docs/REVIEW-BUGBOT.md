## التفاصيل

### 1. نظرة مكتبّي تخفي أخطاء الـ API — high

**الملف:** `frontend/src/app/maktabi/(lawha)/page.tsx` (حوالي السطور 70–79)

كل جلب لبيانات لوحة التحكم ملفوف بـ `.catch(() => [])`، فأخطاء المصادقة و503 وغيرها تُعامل كاستجابة ناجحة فارغة. الصفحة تخرج من حالة التحميل، تتخطى شريط الخطأ، وقد تعرض حالة فارغة للمؤلف فقط أو تخفي أقسام المحكّم/المحرر رغم وجود مهام حقيقية.

### 2. تسجيل الدخول الإداري يسقط الروابط العميقة — medium

**الملف:** `frontend/src/middleware.ts` (حوالي السطور 7–18)

إضافة `/admin(.*)` إلى المسارات المحمية ترسل المستخدم غير المصادق إلى `/tawajjuh` دون معلمة `next` للعودة، بخلاف سلوك `/daawa`. بعد تسجيل الدخول لا يُعاد المستخدم إلى صفحة الإدارة التي طلبها (مثل `/admin/maqalat/[id]`).

### 3. إحصائيات الإدارة تستثني المقالات المنشورة — medium

**الملف:** `frontend/src/app/admin/page.tsx` (حوالي السطور 61–67)

بطاقات الملخص في النظرة العامة تعدّ فقط حالات الإصدار `draft`، `submitted`/`under_review`، `accepted`، و`rejected`. المقالات ذات الحالة `published` مستثناة من كل بطاقة، فتختفي المخطوطات المنشورة من إجماليات الإدارة.

### 4. الشريط الجانبي يخفي الأدوار عند أخطاء الـ API — medium

**الملف:** `frontend/src/components/dashboard/dashboard-shell.tsx` (حوالي السطور 26–34)

فشل `listMyAssignments` و`listEditorArticles` يُحوَّل إلى مصفوفات فارغة، فتُحذف أقسام تنقّل المحكّم والمحرر كلما فشلت تلك الاستدعاءات، حتى لو كان للمستخدم مهام نشطة.
